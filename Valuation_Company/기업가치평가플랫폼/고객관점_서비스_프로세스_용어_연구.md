## 1. 연구 배경

### 1.1 목적
기업가치평가 서비스의 13단계 프로세스를 **고객사(평가 의뢰 기업)** 관점에서 명확하고 친화적으로 표현

### 1.2 사용 맥락
- **위치**: 고객 사이드바 진행 상황 표시
- **대상**: 평가를 의뢰하는 기업 담당자
- **목적**: 현재 진행 단계 한눈에 파악

### 1.3 현재 단계 (초안)
1. 안내문 보기
2. 평가 요청하기
3. 관리자 승인
4. 회사 자료 제출하기
5. AI 자료 수집
6. 평가 실행하기
7. 평가보고서 초안 생성
8. 공인회계사 검토
9. 평가보고서 수정안 작성
10. 회사 수정 요청
11. 평가보고서 최종안 작성
12. 대금 결제
13. 평가보고서 제공

## 2. 유사 서비스 벤치마크

### 2.1 금융 서비스 (대출 심사)

#### 프로세스 단계
| 단계 | 표현 | 특징 |
|------|------|------|
| 1 | **대출 신청** | 능동형 (고객 행동) |
| 2 | **심사 중** | 진행형 (대기) |
| 3 | **심사 결과 확인** | 완료형 (확인) |
| 4 | **약정 체결** | 능동형 (고객 행동) |
| 5 | **대출 실행** | 완료형 (제공) |

#### 핵심 특징
- **고객 행동**: "~신청", "~확인", "~체결"
- **기관 처리**: "심사 중", "검토 중"
- **완료/제공**: "~완료", "~실행"

**출처**: [KB국민은행 대출신청](https://obank.kbstar.com/quics?page=C103514), [우리은행 대출상담](https://spot.wooribank.com/pot/Dream?withyou=POLON0038)

### 2.2 컨설팅 서비스

#### 프로젝트 관리 5단계 (PMBOK 기준)
| 단계 | 영문 | 한글 표현 |
|------|------|----------|
| 1 | Initiating | **착수** |
| 2 | Planning | **계획 수립** |
| 3 | Executing | **실행 중** |
| 4 | Performance | **검토** |
| 5 | Closing | **완료** |

#### 고객 소통 용어
- **RFP**: 제안 요청서 (고객이 요청)
- **Kick-off**: 프로젝트 시작 회의
- **Delivery**: 결과물 전달
- **QA (Quality Assurance)**: 품질 검증

#### 핵심 특징
- **시작**: "착수", "요청"
- **진행**: "~중", "~진행"
- **완료**: "전달", "완료"

**출처**: [프로젝트 관리 5단계](https://slack.com/intl/ko-kr/blog/productivity/project-management-step), [Asana 프로젝트 관리](https://asana.com/ko/resources/project-management-phases)

### 2.3 법무/세무 서비스

#### 법인설립 프로세스 (법무법인)
| 단계 | 표현 |
|------|------|
| 1 | **상담 신청** |
| 2 | **서류 준비** |
| 3 | **검토 진행** |
| 4 | **등기 신청** |
| 5 | **완료 통지** |

#### 핵심 특징
- 고객이 **신청/준비/확인**하는 단계 명확히 구분
- 전문가가 처리하는 단계는 **진행/검토** 표현

**출처**: [법인설립지원센터](https://k-startbiz.org/), [법무법인 대륜](https://www.daeryunlaw-comp.com/advice)

## 3. 고객 관점 용어 원칙

### 3.1 행위 주체별 표현 원칙

```
┌─────────────────────────────────────────────────────┐
│  고객 능동 단계    →  "~하기", "~신청", "~제출"     │
│  고객 수동 대기    →  "~중", "~진행 중"             │
│  고객 확인 단계    →  "~확인", "~검토"              │
│  서비스 제공 완료  →  "~완료", "~제공"              │
└─────────────────────────────────────────────────────┘
```

### 3.2 "회사" vs "기업" vs 생략

| 맥락 | 권장 표현 | 이유 |
|------|----------|------|
| **고객이 행동하는 단계** | 생략 또는 "자료" | 간결성 (예: "자료 제출하기") |
| **고객을 지칭할 때** | "회사" | 친근함 (예: "회사 수정 요청") |
| **공식 문서/보고서** | "기업" | 전문성 (예: "기업가치평가보고서") |

**결론**: 프로세스 단계명에서는 **생략 또는 "자료"** 사용

### 3.3 어미 선택 가이드

| 상황 | 권장 어미 | 예시 | 사용 맥락 |
|------|----------|------|----------|
| **고객 행동** | ~하기 | 신청하기, 제출하기 | 고객이 직접 수행 |
| **진행 중** | ~중 | 심사 중, 검토 중 | 대기 상태 |
| **확인 단계** | 확인 | 결과 확인, 초안 확인 | 고객이 보는 단계 |
| **완료 제공** | 제공, 완료 | 보고서 제공, 평가 완료 | 최종 단계 |

### 3.4 단계별 세부 원칙

#### A. 고객이 능동적으로 하는 단계
```
✅ "평가 요청하기"  (O)
✅ "자료 제출하기"  (O)
✅ "수정 요청하기"  (O)

❌ "평가 요청 진행" (X) - 고객 행동에는 "진행" 어색
❌ "자료 제출 완료" (X) - 완료는 시스템이 판단
```

#### B. 고객이 수동적으로 기다리는 단계
```
✅ "심사 중"         (O)
✅ "평가 진행 중"    (O)
✅ "검토 중"         (O)

❌ "심사하기"        (X) - 고객이 하는 게 아님
❌ "평가 실행하기"   (X) - 전문가가 하는 작업
```

#### C. 고객이 확인하는 단계
```
✅ "초안 확인"       (O)
✅ "결과 확인"       (O)

❌ "초안 생성"       (X) - 확인 대상이 아닌 행위
```

#### D. 완료/전달 단계
```
✅ "보고서 제공"     (O)
✅ "평가 완료"       (O)

❌ "보고서 수령하기" (X) - 불필요하게 길음
```

## 5. 단계별 상세 설명

### 5.1 추천안 (개선안 A) 각 단계 분석

#### 1단계: 서비스 안내
```
고객 행동: 서비스 내용 확인
표현 원칙: 명사형 (간결)
대안: "안내문 확인", "서비스 소개"
```

#### 2단계: 평가 신청
```
고객 행동: 평가 요청서 작성 및 제출
표현 원칙: 능동형 명사
왜 "신청"?: 금융권 관례 ("대출 신청", "서비스 신청")
대안: "평가 요청", "신청하기"
```

#### 3단계: 승인 대기 중
```
고객 행동: 관리자 승인 대기
표현 원칙: 진행형 (~중)
왜 "대기 중"?: 고객이 기다리는 상태 명확
대안: "심사 중", "검토 중"
```

#### 4단계: 자료 제출
```
고객 행동: 평가에 필요한 자료 업로드
표현 원칙: 능동형 명사
왜 "자료"?: "회사 자료"보다 간결, 의미 명확
대안: "자료 업로드", "서류 제출"
```

#### 5단계: 자료 수집 중
```
고객 행동: AI의 추가 자료 수집 대기
표현 원칙: 진행형 (~중)
왜 "자료 수집"?: "AI"보다 수집 행위에 집중
대안: "추가 자료 수집 중", "데이터 수집 중"
```

#### 6단계: 평가 진행 중
```
고객 행동: AI의 평가 실행 대기
표현 원칙: 진행형 (~중)
왜 "진행 중"?: 실시간 처리 중임을 표현
대안: "평가 중", "분석 중"
```

#### 7단계: 초안 확인
```
고객 행동: AI 생성 보고서 초안 검토
표현 원칙: 확인형
왜 "확인"?: 고객이 보고 검토하는 단계
대안: "보고서 초안 확인", "초안 검토"
```

#### 8단계: 전문가 검토 중
```
고객 행동: 회계사의 검토 대기
표현 원칙: 진행형 (~중)
왜 "전문가"?: 공인회계사보다 간결, 신뢰감
대안: "회계사 검토 중", "검증 중"
```

#### 9단계: 수정안 확인
```
고객 행동: 회계사 수정 반영본 검토
표현 원칙: 확인형
왜 "수정안"?: 초안 대비 개선본 의미
대안: "보고서 수정안 확인"
```

#### 10단계: 수정 요청
```
고객 행동: 추가 수정사항 요청
표현 원칙: 능동형 명사
왜 "수정 요청"?: 고객의 능동적 피드백
대안: "수정사항 요청", "피드백 제출"
```

#### 11단계: 최종안 확인
```
고객 행동: 최종 보고서 검토 및 승인
표현 원칙: 확인형
왜 "최종안"?: 프로세스의 마지막 검토 단계
대안: "최종 보고서 확인"
```

#### 12단계: 결제하기
```
고객 행동: 평가 대금 결제
표현 원칙: 능동형 (~하기)
왜 "결제하기"?: 명확한 행동 유도 (CTA)
대안: "결제", "대금 결제"
```

#### 13단계: 보고서 수령
```
고객 행동: 최종 보고서 다운로드/수령
표현 원칙: 완료형
왜 "수령"?: 서비스 완료 + 결과물 획득
대안: "완료", "보고서 제공"
```

### 5.2 고객 여정 단계 분류

```
┌─────────────────────────────────────────────────────┐
│  Phase 1: 시작 (고객 주도)                           │
│  → 1. 서비스 안내                                    │
│  → 2. 평가 신청                                      │
│  → 3. 승인 대기 중                                   │
│  → 4. 자료 제출                                      │
├─────────────────────────────────────────────────────┤
│  Phase 2: 평가 (시스템 주도, 고객 대기)              │
│  → 5. 자료 수집 중                                   │
│  → 6. 평가 진행 중                                   │
│  → 7. 초안 확인                                      │
├─────────────────────────────────────────────────────┤
│  Phase 3: 검증 (전문가 주도, 고객 협업)              │
│  → 8. 전문가 검토 중                                 │
│  → 9. 수정안 확인                                    │
│  → 10. 수정 요청                                     │
```
