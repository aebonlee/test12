## 🚀 작업 진행 순서

### 1단계: 파일 배치
- **참고자료** 폴더: 관련 논문, 기술 문서, 유사 특허, 플랫폼 개발 계획서 등
- **도면** 폴더: 제11도, 제12도 등 새로 만들 도면 파일들
- **특허출원서_수정전버전** 폴더: 기존 특허출원서 원본 보관
- **검토의견** 폴더: 수정 가이드 문서 정리
- **특허출원서_수정진행버전** 폴더: 단계별 수정 작업 진행 파일들
- **특허출원서_최종버전** 폴더: 완성된 특허출원서 최종본

## 🔄 진행 방식

### 🎨 도면 작업: 멀티 에이전트 병렬 처리
**여러 도면 전문 서브 에이전트를 투입하여 병렬 작업**

**⚠️ 중요: 기존 도면은 참조만, 완전히 새로운 도면 제작 필요**
- 기존 특허출원서의 도면은 참조용으로만 활용
- 모든 도면을 새롭게 제작해야 함

**도면별 전문 에이전트 배치:**
- **에이전트 A**: 제11도 - 시스템 전체 구성도 새로 제작
- **에이전트 B**: 제12도 - 데이터 플로우 다이어그램 새로 제작  
- **에이전트 C**: 추가 도면 - 사용자 인터페이스 설계도 새로 제작
- **에이전트 D**: 보조 도면 - 기술적 상세 구조도 새로 제작

**병렬 처리 장점:**
- 동시 작업으로 시간 단축
- 각 도면별 전문성 집중
- 독립적 품질 관리 가능
- 완전히 새로운 디자인으로 특허 강화

### 📝 특허출원서 작업: 멀티 롤 순차 처리
**역할을 바꿔가며 각 섹션별 전문가 모드로 작업**

**1단계: 전문가 모드 작성**
- 🏛️ **명세서 전문 변리사 모드**: 기술분야, 배경기술, 발명의 내용 작성
- ⚖️ **청구항 전문 변리사 모드**: 독립청구항, 종속청구항 정교화
- 📋 **도면설명 전문 변리사 모드**: 도면과 연계된 상세 설명 작성
- 🔍 **요약서 전문 변리사 모드**: 발명의 요약 및 핵심 정리

**2단계: 검토자 모드**
- 📊 **기술 검토자 모드**: 기술적 정확성, 논리성 검증
- 📏 **법률 검토자 모드**: 특허법 요건 준수 여부 점검
- 🎯 **전략 검토자 모드**: 권리범위 최적화 검토

**3단계: 비판자 모드**
- 🔨 **심사관 시각 비판자**: 거절 이유 발굴 및 약점 분석
- ⚔️ **경쟁사 시각 비판자**: 회피 설계 가능성 검토
- 🚫 **무효심판 시각 비판자**: 선행기술 대비 신규성 검증

**4단계: 수정자 모드**
- 🔧 **종합 수정자**: 모든 의견 취합하여 최종 수정
- ✅ **품질 관리자**: 전체 일관성 및 완성도 점검

## ⚠️ 작업 시 주의사항

### 성급함 방지 원칙
- 복잡한 특허 수정작업이므로 **단계별 검토** 필수
- 각 수정사항에 대해 **법적 타당성** 먼저 확인
- **전체 일관성** 유지하며 부분별 수정 진행

### 품질 관리 체크포인트
- [ ] 청구항 간 논리적 연결성 확인
- [ ] 기술적 명세의 정확성 검토
- [ ] 도면과 설명의 일치성 점검
- [ ] 특허법 요건 준수 여부 확인

## 📋 작업 전 확인사항

특허 수정 작업을 시작하기 전에 다음을 확인해주세요:

1. **수정 범위**: 어느 부분까지 수정이 필요한지 구체적 확인
2. **기술적 정확성**: 변경할 기술 내용의 정확성 검토
3. **법적 요건**: 특허법상 요구사항 준수 여부
4. **우선순위**: 가장 중요한 수정사항부터 순서 정리

## 🔄 진행 방식

### 점진적 수정 접근법
1. **10% 완성**: 주요 수정 방향 설정 및 확인
2. **30% 완성**: 핵심 청구항 수정 완료
3. **50% 완성**: 기술명세 보완 및 도면 연동
4. **80% 완성**: 전체 일관성 검토 및 조정
5. **100% 완성**: 최종 검토 및 완성본 제출

### 각 단계별 검토 포인트
- 수정 내용의 **기술적 타당성**
- 청구항 간 **논리적 연결성**
- 전체 문서의 **일관성 유지**
- 특허 요건 **법적 준수성**
