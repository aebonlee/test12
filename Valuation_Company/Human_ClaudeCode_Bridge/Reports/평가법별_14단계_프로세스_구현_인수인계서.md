## 작업 배경

### 문제 인식
- 평가법 가이드 페이지(5개)의 사이드바에 8단계 프로세스가 있으나, 실제 페이지와 연결되지 않음
- 평가법별로 다른 포털/결과 페이지로 연결되어야 하나 현재는 고정 링크
- 시스템 구성도(SVG)와 실제 프로세스가 불일치

### 목표
1. 사이드바 프로세스를 실제 시스템 구성도에 맞게 14단계로 재구성
2. 평가법별로 동적으로 올바른 페이지로 연결
3. 누락된 페이지 신규 생성

## 주요 결정사항

### 1. 용어 및 명칭
- **평가법 영문명**: Intrinsic Value Method (본질가치평가법)
- **프로젝트 코드**: IV (본질가치평가법)
- **고객 관점 용어**: 능동형("신청", "제출") vs 대기형("~중")
- **단계 수**: 14단계 (최종 확정)

### 2. 보고서 관련
- **보고서 종류 선택 시점**: 2단계(평가 신청)
- **보고서 금액 표시**: 1단계(서비스 안내)에서 안내
- **보고서 발행 소요 시간**: 24시간 이내
- **수령 방법**: 2단계에서 선택 (이메일/우편)

### 3. 데이터 용어 구분
- **평가 기초자료**: 고객이 제출하는 재무제표 등
- **데이터**: AI가 자동 수집하는 외부 데이터

### 4. 프로세스 흐름
- **평가 순서**: 평가 진행 → 공인회계사 검토 → 초안 생성 → 초안 확인
- **수정 프로세스**: 초안 확인 → 수정 요청 → 최종안 작성 → 최종안 확인
- **수정안 확인 단계 제거**: 고객은 최종안만 확인

### 5. 시스템 구성도 기반
- **기준 문서**: `기업가치평가_시스템_구성도.svg`
- **처리 시간**: 총 3-5시간 (AI 자동화로 단축)
