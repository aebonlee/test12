## 8단계 고객 프로세스 개요

```
┌─────────────────────────────────────────────────────────────┐
│                    고객 여정 (Customer Journey)              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [1] 안내문 보기 ─────── 평가방법 이해, 절차 안내           │
│       ↓                                                     │
│  [2] 평가 요청하기 ───── 회사 정보 입력, 평가방법 선택       │
│       ↓                                                     │
│  [3] 관리자 승인 대기 ── 관리자가 요청 검토 후 승인/반려     │
│       ↓                                                     │
│  [4] 자료 제출하기 ───── 필요서류 업로드, 체크리스트 확인    │
│       ↓                                                     │
│  [5] 평가 결과 미리보기 ─ AI 분석 결과 확인, 시뮬레이션      │
│       ↓                                                     │
│  [6] 수정 요청하기 ───── 가정 변경, 재계산 요청             │
│       ↓                                                     │
│  [7] 최종 결과 보기 ──── 확정된 평가 결과 확인              │
│       ↓                                                     │
│  [8] 보고서 발행 요청 ── 공식 보고서 발행 신청              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

## 단계별 상세

### Step 1: 안내문 보기

| 항목 | 내용 |
|------|------|
| 목적 | 평가방법 이해 |
| 페이지 | `/guide/{method}` |
| 콘텐츠 | 교육자료 기반 (03_평가방법 폴더) |
| 액션 | [이 방법으로 평가 요청하기] 버튼 |

### Step 2: 평가 요청하기

| 항목 | 내용 |
|------|------|
| 목적 | 평가 요청 접수 |
| 페이지 | `/request/new` |
| 입력 | 회사명, 사업자번호, 평가목적, 평가방법, 희망완료일 |
| 결과 | 프로젝트 ID 생성 |

### Step 3: 관리자 승인 대기

| 항목 | 내용 |
|------|------|
| 목적 | 요청 검토 및 승인 |
| 페이지 | `/project/{id}` (상태 표시) |
| 관리자 | `/admin/requests` 에서 승인/반려 |
| 알림 | 승인 시 이메일 발송 |

### Step 4: 자료 제출하기

| 항목 | 내용 |
|------|------|
| 목적 | 평가 필요서류 수집 |
| 페이지 | `/project/{id}/documents` |
| 기능 | 체크리스트, 파일 업로드, 임시저장 |
| 콘텐츠 | 평가방법별 필요서류 목록 |

### Step 5: 평가 결과 미리보기

| 항목 | 내용 |
|------|------|
| 목적 | AI 평가 결과 확인 |
| 페이지 | `/project/{id}/preview` |
| 기능 | 결과 표시, 민감도 분석, 시뮬레이션 |
| 엔진 | 검증된 평가 엔진 사용 |

### Step 6: 수정 요청하기

| 항목 | 내용 |
|------|------|
| 목적 | 가정/파라미터 변경 요청 |
| 페이지 | `/project/{id}/revision` |
| 입력 | 수정 유형, 상세 내용, 근거 자료 |
| 반복 | Step 5 → Step 6 반복 가능 |

### Step 7: 최종 결과 보기

| 항목 | 내용 |
|------|------|
| 목적 | 확정된 평가 결과 확인 |
| 페이지 | `/project/{id}/result` |
| 기능 | 최종 가치, 상세 보고서 미리보기 |

### Step 8: 보고서 발행 요청

| 항목 | 내용 |
|------|------|
| 목적 | 공식 보고서 발행 |
| 페이지 | `/project/{id}/report` |
| 옵션 | 간이/정식/감정평가서, PDF/인쇄본 |

## 페이지 URL 구조

| Step | 페이지 | URL |
|------|--------|-----|
| 0 | 로그인 | `/login` |
| 0 | 회원가입 | `/register` |
| 0 | 대시보드 | `/dashboard` |
| 1 | 평가방법 안내 | `/guide/{method}` |
| 2 | 평가 요청 | `/request/new` |
| 3 | 승인 대기 | `/project/{id}` |
| 4 | 자료 제출 | `/project/{id}/documents` |
| 5 | 결과 미리보기 | `/project/{id}/preview` |
| 6 | 수정 요청 | `/project/{id}/revision` |
| 7 | 최종 결과 | `/project/{id}/result` |
| 8 | 보고서 발행 | `/project/{id}/report` |
| - | 관리자 포털 | `/admin/*` |

## 프로젝트 상태 (Status)

```
REQUESTED → APPROVED → DOCUMENTS_SUBMITTED → ANALYSIS_COMPLETE
         → REVISION_REQUESTED → FINALIZED → REPORT_ISSUED
```

| 상태 | 설명 | 고객 액션 |
|------|------|----------|
| REQUESTED | 평가 요청됨 | 대기 |
| APPROVED | 관리자 승인 | 자료 제출 |
| DOCUMENTS_SUBMITTED | 자료 제출 완료 | 대기 |
| ANALYSIS_COMPLETE | 분석 완료 | 결과 확인 |
| REVISION_REQUESTED | 수정 요청 중 | 대기 |
| FINALIZED | 최종 확정 | 보고서 요청 |
| REPORT_ISSUED | 보고서 발행 | 완료 |
