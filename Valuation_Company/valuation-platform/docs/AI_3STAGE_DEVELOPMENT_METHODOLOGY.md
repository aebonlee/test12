# AI 시대의 3단계 개발 방법론: 목업 → 프로토타입 → 프로덕션

## 개요

AI를 활용한 소프트웨어 개발에서 목업과 프로덕션의 경계가 사라지고 있습니다. 과거에는 목업과 프로덕션 간 품질 차이가 컸지만, AI 시대에는 목업도 프로덕션급으로 작동합니다. 각 단계는 단순히 디자인이나 관리 방법론만 추가되는 것이 아니라, **기능도 점진적으로 발견되고 추가**됩니다.

---

## 3단계 개발 프로세스

```
┌─────────────┐
│   목업      │  핵심 기능 구현 + 구조 설계
│  (Mock-up)  │
└──────┬──────┘
       │ + 디자인 시스템 + 누락 기능 발굴
       ↓
┌─────────────┐
│ 프로토타입   │  목업 + 세련된 디자인 + 추가 기능
│ (Prototype)  │
└──────┬──────┘
       │ + 체계적 개발 방법론 + 추가 기능
       ↓
┌─────────────┐
│  프로덕션    │  프로토타입 + 유지보수성 + 완전한 기능
│ (Production) │
└─────────────┘
```

---

## 1단계: 목업 (Mock-up)

### 정의
실제 제품의 **핵심 기능과 구조**를 보여주는 **작동하는 모형**

### 특징
- ✅ **핵심 기능 구현** (전체 기능의 70~80%)
- ✅ **주요 흐름 작동**
- ❌ **디자인 시스템 없음** - "막 만들기"
- ❌ **체계적 관리 방법론 없음**
- ❌ **세부 기능 누락 가능** (다음 단계에서 발굴)

### 목적
- 아이디어 검증
- 기술적 실현 가능성 확인
- 핵심 구조 설계 및 API 인터페이스 정의
- 주요 플로우가 작동하는 데모 제작

### 산출물 예시
- 핵심 API 엔드포인트
- 기본 UI/UX (디자인 무시)
- 더미 데이터로 작동하는 주요 플로우
- 데이터베이스 스키마

### 기능 범위
```
✅ 구현하는 것:
- 핵심 사용자 시나리오
- 주요 CRUD 기능
- 기본 인증/권한
- 핵심 비즈니스 로직

❌ 미구현 (다음 단계로 미룸):
- 세부 에러 처리
- 엣지 케이스
- 부가 기능
- 관리자 세부 기능
- 통계/리포트
```

---

## 2단계: 프로토타입 (Prototype)

### 정의
목업에 **디자인 시스템을 적용**하고 **누락된 기능을 발굴/추가**한 버전

### 특징
- ✅ **목업의 모든 기능 유지**
- ✅ **디자인 시스템 적용** (색상, 타이포그래피, 컴포넌트 등)
- ✅ **누락 기능 발굴 및 추가** ⭐ 중요!
- ✅ **추가 메뉴/화면 발견 및 구현**
- ✅ **일관된 UI/UX**
- ❌ **체계적 관리 방법론은 아직 미적용**

### 목적
- 목업 사용 중 **발견된 누락 기능 보완**
- 사용자 테스트
- 투자 유치용 데모
- 베타 서비스 출시
- 디자인 완성도 확보

### 누락 기능 발굴 프로세스

#### 1. 목업 사용 중 발견
```
사용자: "로그인은 되는데 비밀번호 찾기가 없네?"
     → 비밀번호 찾기 기능 추가

사용자: "관리자 페이지에 사용자 목록만 있고 검색이 없네?"
     → 검색/필터 기능 추가

사용자: "데이터를 Excel로 내보내기 하고 싶은데?"
     → Export 기능 추가
```

#### 2. 시나리오 재검토
```
목업 제작 시 놓친 사용자 시나리오:
- 회원 탈퇴
- 계정 정지/복구
- 대량 데이터 일괄 처리
- 알림 설정
- 프로필 이미지 업로드
```

#### 3. 화면별 체크리스트
```
각 화면마다:
□ 목록 조회
□ 상세 조회
□ 생성
□ 수정
□ 삭제
□ 검색
□ 필터링
□ 정렬
□ 페이지네이션
□ Export

→ 누락된 항목 발견 및 추가
```

#### 4. 추가 메뉴 발견
```
목업 단계:
- 대시보드
- 프로젝트 관리
- 설정

프로토타입 단계에서 추가:
- 통계/리포트 ← 새로 발견
- 알림 센터 ← 새로 발견
- 사용자 활동 로그 ← 새로 발견
- FAQ/고객지원 ← 새로 발견
```

### 디자인 시스템이란?
- **색상 팔레트**: Primary, Secondary, Accent 등
- **타이포그래피**: 폰트, 크기, 행간 규칙
- **컴포넌트 라이브러리**: 버튼, 입력창, 카드 등 재사용 가능한 UI 요소
- **간격/여백 시스템**: 4px, 8px, 16px 등 일관된 spacing
- **아이콘 세트**: 통일된 스타일의 아이콘

### 산출물 예시
- Material-UI, Tailwind CSS 등 적용된 UI
- Figma/Sketch 디자인 시스템 문서
- 스타일 가이드
- **기능 추가 목록 문서** ⭐
- 브랜드 일관성이 적용된 완성도 높은 제품

## 3단계: 프로덕션 (Production)

### 정의
프로토타입에 **체계적 개발 방법론을 적용**하고 **추가 기능을 구현**한 최종 제품

### 특징
- ✅ **프로토타입의 모든 기능/디자인 유지**
- ✅ **체계적 개발 방법론 적용**
- ✅ **운영 중 발견되는 추가 기능 구현** ⭐
- ✅ **유지보수 용이**
- ✅ **확장 가능한 구조**

### 목적
- 실제 서비스 운영
- 장기적 유지보수
- 팀 협업 효율화
- 코드 품질 관리

### 프로덕션 단계 추가 기능

```
실제 운영 중 발견:
- 성능 모니터링 대시보드
- 에러 추적 시스템
- 백업/복구 기능
- A/B 테스트 기능
- 다국어 지원
- 접근성 개선
- SEO 최적화
- 보안 강화 (2FA 등)
```

### 체계적 개발 방법론이란?

#### 1. 코드 구조화
```
명확한 폴더 구조
- /components
- /services
- /utils
- /hooks
- /types
등 역할별 분리
```

#### 2. 문서화
```
- API 명세서
- 코드 주석
- README
- 아키텍처 문서
```

#### 3. 테스트
```
- 단위 테스트 (Unit Test)
- 통합 테스트 (Integration Test)
- E2E 테스트
- 테스트 커버리지 관리
```

#### 4. 버전 관리
```
- Git Flow
- 브랜치 전략
- Commit 규칙
- Semantic Versioning
```

#### 5. CI/CD
```
- 자동 빌드
- 자동 테스트
- 자동 배포
- 롤백 전략
```

#### 6. 코드 리뷰
```
- PR (Pull Request) 프로세스
- 코드 리뷰 체크리스트
- 코딩 컨벤션
```

#### 7. 모니터링/로깅
```
- 에러 추적 (Sentry)
- 성능 모니터링
- 사용자 행동 분석
- 로그 관리
```

## 각 단계별 비교표

| 항목 | 목업 | 프로토타입 | 프로덕션 |
|------|------|-----------|---------|
| **기능 완성도** | 70~80% | 85~95% | 100% |
| **기능 발굴** | ❌ 핵심만 | ✅ 누락 발굴 | ✅ 추가 발굴 |
| **디자인 시스템** | ❌ 없음 | ✅ 적용 | ✅ 적용 |
| **체계적 관리** | ❌ 없음 | ❌ 없음 | ✅ 적용 |
| **개발 속도** | ⚡ 매우 빠름 | 🚀 빠름 | 🐢 느림 |
| **유지보수성** | 🔻 낮음 | 📊 중간 | 🔝 높음 |
| **코드 품질** | 📝 기본 | 📝 기본 | ✨ 높음 |
| **용도** | 검증/데모 | 베타 서비스 | 정식 서비스 |

## 왜 3단계로 나누는가?

### 1. 리스크 최소화
목업으로 빠르게 검증 → 실패 시 적은 비용으로 포기 가능

### 2. 투자 효율성
```
목업: 1주일, 비용 低
  → 실패 시 손실 적음

처음부터 프로덕션: 3개월, 비용 高
  → 실패 시 손실 큼
```

### 3. 점진적 완성도 향상
```
목업 (핵심 기능 70%)
  → 프로토타입 (디자인 + 누락 기능 발굴 → 90%)
  → 프로덕션 (체계 + 추가 기능 → 100%)
```

### 4. 기능 발굴의 점진성
```
목업 사용 → "아, 이 기능도 필요하네" 발견
프로토타입 사용 → "이것도 있으면 좋겠네" 발견
프로덕션 운영 → "사용자가 이것도 원하네" 발견
```

### 다음 단계: 프로토타입

**추가할 것:**

- 컴포넌트 라이브러리 구축
- 반응형 디자인 적용

#### 2. 누락 기능 발굴 및 추가 ⭐
```
목업 사용 중 발견할 기능들:

□ 인증/계정
  - 비밀번호 찾기
  - 이메일 인증
  - 프로필 수정
  - 회원 탈퇴

  - 고급 검색 (날짜, 상태, 평가법별)

  - 파일 미리보기 (PDF, Excel)

□ 보고서
  - 초안 버전 관리 (v1, v2, v3...)
  - 수정 이력 추적

□ 알림

  - 알림 설정 관리

□ 통계/리포트
  - 프로젝트 통계 대시보드
  - 회계사별 처리 현황
  - 평가법별 분포
  - 월별 매출 현황

□ 관리자 기능
  - 사용자 관리
  - 권한 관리
  - 시스템 설정
  - 감사 로그

□ 고객지원
  - FAQ
  - 1:1 문의
  - 공지사항
  - 이용 가이드
```

```

- 대시보드
- 프로젝트
- 설정

- 통계 ← 새로 발견
- 알림 ← 새로 발견
```

## 핵심 인사이트

### 기능은 단계별로 발견된다

```
목업 단계:
"이 정도면 작동하겠지" (70~80% 기능)

프로토타입 단계:
"아, 이것도 필요하네!" (누락 기능 발견)
→ 검색, 필터, Export, 알림 등 추가

프로덕션 단계:
"사용자가 이것도 원하네!" (추가 기능 발견)
→ 고급 통계, 통합, 자동화 등 추가
```

### 모든 기능을 처음부터 구현하지 않는 이유

1. **시간 낭비 방지**: 안 쓰이는 기능 만들 수 있음
2. **빠른 검증**: 핵심만 먼저 검증
3. **사용자 피드백**: 실제 사용 중 필요 기능 발견
4. **점진적 완성**: 단계별로 안정적 추가

### AI 시대의 차이점

**과거 (AI 없음):**
```
목업: 껍데기, 작동 안 함
  ↓ 막대한 시간/비용
프로덕션: 실제 작동

→ 목업은 "보여주기용"
```

**현재 (AI 활용):**
```
목업: 실제 작동, 프로덕션급 기능
  ↓ 디자인/체계만 추가
프로덕션: 목업 + α

→ 목업도 "실제 사용 가능"
```

## 적용 가이드

### 목업 → 프로토타입 전환 타이밍
- ✅ 목업으로 핵심 기능 검증 완료
- ✅ 사용자가 "이것도 있으면 좋겠다" 피드백 시작
- ✅ 베타 서비스 출시 계획
- ✅ 투자 유치 준비

### 프로토타입 → 프로덕션 전환 타이밍
- ✅ 프로토타입으로 대부분 기능 검증 완료
- ✅ 정식 서비스 출시 확정
- ✅ 팀 확장 예정
- ✅ 장기 운영 계획
