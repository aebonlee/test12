# /evaluate Command

콘텐츠 품질을 평가하고 100점 만점의 종합 점수를 제공하는 명령어입니다.

## 사용법
```
/evaluate <파일_경로_또는_디렉토리>
```

## 설명
이 명령어는 작성된 콘텐츠를 5가지 기준으로 평가합니다 (각 20점):
- **기술적 정확성** (Technical Accuracy): 정보의 정확성과 최신성
- **가독성** (Readability): 읽기 쉬움과 이해도
- **구조 및 구성** (Structure & Organization): 논리적 흐름과 체계적 배치
- **완성도** (Completeness): 주제의 포괄적 다룸
- **유용성** (Usefulness): 실무 적용 가능성과 실용적 가치

## 파라미터
- `$ARGUMENTS`: 평가할 파일 경로 또는 디렉토리

## 필요한 도구
- Read
- Glob
- Grep
- Task (평가 수행용)

📝 평가 대상: [파일명 또는 콘텐츠 제목]

📈 세부 평가:
• 기술적 정확성: XX/20점

## 구현 로직
1. 지정된 파일 또는 디렉토리 읽기
2. 5가지 평가 기준에 따라 콘텐츠 분석
3. 각 기준별 점수 계산
4. 종합적인 평가 피드백 생성
5. 건설적인 제안과 함께 형식화된 결과 출력

## 사용 예시
```bash
# 단일 파일 평가
/evaluate "G:\내 드라이브\Content\AI실전활용마스터\Part 3 웹개발 기초지식\17편-SEO와-웹-접근성.md"

# 디렉토리 전체 평가
/evaluate "G:\내 드라이브\Content\AI실전활용마스터\Part 3 웹개발 기초지식"

# 현재 디렉토리 평가
/evaluate .
```

## 평가 기준 상세 설명

### 1. 기술적 정확성 (20점)
- 정보의 정확도
- 최신 기술 트렌드 반영
- 용어 사용의 적절성
- 예제 코드의 정확성

### 2. 가독성 (20점)
- 문장의 명확성
- 단락 구성의 적절성
- 전문 용어 설명의 충실도
- 흐름의 자연스러움

### 3. 구조 및 구성 (20점)
- 논리적 순서
- 섹션 구분의 명확성
- 제목과 부제목의 체계
- 전체적인 일관성

### 4. 완성도 (20점)
- 주제의 포괄적 다룸
- 필수 내용 포함 여부
- 예제와 설명의 균형
- 마무리의 완결성

### 5. 유용성 (20점)
- 실무 적용 가능성
- 실습 예제의 실용성
- 학습 목표 달성도
- 참고 자료의 가치
