## 지적재산권 개요

ValueLink의 핵심 경쟁력은 **5개 평가 엔진**과 **14단계 워크플로우 시스템**입니다. 이를 **영업 비밀(Trade Secret)**로 보호하는 전략을 채택합니다.

```
핵심 IP = 5개 평가 엔진 + 워크플로우 시스템
        ↓
보호 방법 = 영업 비밀 (Trade Secret)
        ↓
이유 = 특허 출원 시 기술 공개, 영업 비밀로 장기 보호
```

## IP 보호 전략

### 영업 비밀 (Trade Secret) 선택 이유

#### 특허 vs 영업 비밀 비교

| 항목 | 특허 | 영업 비밀 (선택) |
|------|------|-----------------|
| **보호 기간** | 출원일로부터 20년 | 비밀 유지하는 한 무기한 |
| **공개 여부** | 출원 시 공개 (18개월 후) | 비공개 유지 |
| **보호 범위** | 명세서 기재 범위 | 실제 사용 범위 |
| **비용** | 출원 + 유지 비용 (수백만원) | 비밀 유지 비용 (낮음) |
| **경쟁사 대응** | 우회 설계 가능 | 비밀 유지 시 복제 어려움 |

**ValueLink 선택**: 영업 비밀

**이유**:
1. **알고리즘 특성**: 소프트웨어 알고리즘은 특허 인정 어려움
2. **공개 리스크**: 특허 출원 시 핵심 로직 공개 → 경쟁사 모방 용이
3. **장기 보호**: 영업 비밀로 무기한 보호 가능
4. **비용 효율**: 특허 출원/유지 비용 절감

### 영업 비밀 보호 조치

#### 1. 법적 보호
```
✅ 직원 NDA (Non-Disclosure Agreement)
- 입사 시 비밀유지 계약 체결
- 퇴사 후 2년간 비밀 유지 의무

✅ 경업 금지 조항 (Non-Compete Clause)
- 동종 업계 이직 제한 (1년)
- 유사 사업 금지 (2년)

✅ 협력사 NDA
- 회계사, 개발 파트너 NDA 체결
```

#### 2. 기술적 보호
```
✅ 소스코드 암호화
- GitHub Private Repository
- 코드 난독화 (Obfuscation)
- 민감 로직은 별도 서버 (API로만 접근)

✅ 접근 제한
- 개발자별 권한 분리
- 핵심 코드는 대표만 접근 가능
- 감사 로그 (Audit Log) 유지

✅ 백업 및 복구
- 매일 백업 (암호화 저장)
- 오프라인 백업 (외장 HDD)
```

#### 3. 조직적 보호
```
✅ Need-to-Know 원칙
- 필요한 사람만 접근 가능
- 전체 시스템은 대표만 파악

✅ 교육
- 직원 대상 IP 보호 교육 (분기 1회)
- 보안 사고 대응 훈련

✅ 모니터링
- 이상 접근 감지
- 코드 유출 모니터링
```

## 데이터 자산 (Data Assets)

### 1. 평가 데이터베이스

**내용**:
- 누적 평가 프로젝트 데이터 (재무제표, 평가 결과)
- 업종별 벤치마크 데이터

**가치**:
- 평가 건수 증가 → 데이터 가치 증가
- 업종별 정확도 향상
- 시장 인사이트 제공 가능

**보호 방법**:
- 데이터베이스 암호화
- 접근 로그 기록
- 정기 백업

### 2. 고객 데이터

**내용**:
- 기업 정보 (비공개 재무 정보)
- 투자자 프로파일
- 매칭 히스토리

**보호 방법**:
- 개인정보보호법 준수
- 데이터 최소화 원칙
- 동의 기반 수집/활용

## 투자 및 M&A 시 IP 가치 평가

### IP 가치 산정

```
영업 비밀 가치 = 개발 비용 + 시장 우위 가치

개발 비용:
- 개발 시간: 3개월 (6,645줄)
- 개발 인건비: 월 500만원 × 3개월 = 1,500만원
- AI API 비용: 300만원
- 총 개발 비용: 1,800만원

시장 우위 가치:
- 평가 엔진으로 인한 연간 매출: 1억원
- 엔진 없이는 서비스 불가 → 대체 불가 자산
- 추정 가치: 매출의 3-5배 = 3-5억원

총 IP 가치: 약 5억원 (3년차 기준)
```

### M&A 시 IP 이전

```
✅ 소스코드 이전 (GitHub 소유권 이전)
✅ 데이터베이스 이전 (암호화 키 포함)
✅ Know-How 이전 (문서화 + 교육)
✅ 상표권 이전 (법적 절차)
```
